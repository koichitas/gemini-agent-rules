# Git / GitHub ルール

-   **メインブランチ運用**: プロジェクトでは `main` ブランチのみを使用します。
-   **mainブランチへの直接プッシュ**: 履歴がGitHubに記録されることを前提として、`main` ブランチへの直接プッシュを許可します。
-   **`--force` コマンドの禁止**: `git push` を実行する際は、いかなる場合でも `--force` コマンドを使用してはなりません。
-   **新規リポジトリの作成**: GitHubアカウント `koichitas` にて、自由に新規リポジトリを作成すること。
-   **コミット後のプッシュ**: コミットを行った後は、その変更を速やかにリモートリポジトリ（GitHub）へプッシュすること。
-   **機密情報の管理**: APIキーやパスワードなどの機密情報が誤ってコミットまたはプッシュされることを防ぐため、`.gitignore` ファイルを適切に管理・更新すること。
