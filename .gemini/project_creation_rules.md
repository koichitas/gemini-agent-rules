# プロジェクト作成ルール

-   **analysis_options.yaml の初期設定**: 新規Flutterプロジェクトを作成する際は、必ずプロジェクトルートに `analysis_options.yaml` ファイルを配置すること。このファイルの初期内容としては、`.gemini/templates/flutter/analysis_options.yaml` にあるテンプレートファイルをコピーして利用すること。
-   **初期リポジリの作成とプッシュ**: 基本プロジェクト（HelloWorldレベル）が作成され、初期動作確認が完了した段階で、速やかにGitHubに新しいリポジトリを作成し、そのプロジェクトをプッシュすること。これにより、早期からバージョン管理を開始し、変更履歴を追跡可能とすること。
